*** timbreRecognition
1. Selectionner CORPUS :Le corpus est l'ensemble des catégories à reconnaitre, les fichiers sons d'apprentissage, les caractéristiques de ces fichiers son, et les modeles statistiques appris sur ces données. Un corpus est contenu dans un répertoire, qui peut être placé n'importe où dans l'ordinateur, indépendamment de l'emplacement du patch. En cliquant sur le bouton 1, on sélectionne ce répertoire : on peut créer un nouveau répertoire vide qui sera rempli au fur et à mesure, ou choisir un corpus pré-existant, sur lequel un certain travail a déjà été commencé.2. Enregistrement : Le sous-patch d'enregistrement permet de définir des nouvelles catégories sonores (des "classes") et d'enregistrer divers exemples sonores pour chacune. En cliquant sur le "+", on ajoute une nouvelle classe, en lui donnant un nom. Celui-ci apparaît ensuite dans le menu déroulant. Il s'agit d'enregistrer plusieurs exemples courts. Par-défaut la durée maximale d'un exemple est fixée à 10 secondes, mais on peut la modifier avec le contrÙle numérique en haut à droite de la fenêtre. Par défaut, la classe "sil" (silence) est définie, avec l'index 0. Il faut quand-même lui fournir des exemples sonores. Lorqu'une classe est sélectionnée dans le menu déroulant, on peut commencer à enregistrer, en cliquant sur REC. L'exemple n'est pas validé tant qu'on n'a pas cliqué sur SAVE. On peut choisir de sauver seulement une partie de l'exemple enregistré en sélectionnant une région de la forme d'onde. Pour être sûr, on peut ré-écouter la région sélectionnée en cliquant sur PLAY. Le niveau "Monitor OUT" contrÙle seulement le niveau d'écoute, l'enregistrement est sauvé à OdB.3. ApprentissageUne fois les exemples validés, les modèles sont appris en cliquant sur Apprentissage. Ils sont normalement sauvés dans le répertoire CORPUS/models. Le déroulement de l'apprentissage est reporté dans la console de Max.4. Reconnaissance  Une fois les modeles appris, la fenêtre de reconnaissance peut être ouverte. Si l'audio est sur ON, la reconnaissance peut commencer. Les histogrammes donnent les valeurs des probabilités de chaque classe dans l'ordre de leur index (chiffre à gauche du menu déroulant dans la fenêtre d'enregistrement.). Le nom de la classe la plus probable est affiché. 

Le "silence boost" dans le patch de reconnaissance est un facteur qui permet de favoriser la classe silence en dessous d'une certaine intensité sonore du son. Quand il est reglé au minimum, l'intensité n'est pas prise en compte pour détecter les silences (c'est juste un timbre parmi les autres). Ca permet de ne pas prendre en compte l'intensité du son dans les classes apprises, mais seulement leur timbre.On peut faire autant d'aller-retours qu'on veut entre les phases 2,3 et 4.*** fxTest.maxpatLe patch fxTest.maxpat est un patch de test d'effets audios (4 effets). La sélection de l'effet est fonction de la classe reconnue. La classe silence "bypass" tous les effets. Les données du patch de reconnaissance sont envoyées via des messages OSC. Cliquer sur "Réception" pour activer la communication entre les 2 patchs (de reconnaissance et d'effets). Le patch fxTest.maxpat nécessite l'installation préalable de jamoma, une série d'objets pour max, gratuite et open source : http://www.jamoma.org .
